# Smoke Checklist (Final Validation)

리팩터링 이후 핵심 동작이 살아있는지 빠르게 확인하기 위한 최종 체크리스트.

## 1) 상단 버튼 동작
- [ ] `업데이트` 버튼 클릭 시 상태 라벨 갱신
- [ ] `현재가` 버튼 클릭 시 편집 모달 오픈
- [ ] `가져오기` 버튼 클릭 후 파일 선택 시 import 동작
- [ ] `초기화` 버튼 클릭 시 초기화 다이얼로그 오픈

## 2) 히스토리 탭
- [ ] `주간`/`월간` 버튼 클릭 시 모드 전환 + 차트 재조회
- [ ] `🔄 새로고침` 버튼 동작
- [ ] 디버그 패널 `닫기` 버튼(`data-action="toggle-hist-debug"`) 동작

## 3) 구글시트 탭
- [ ] URL 입력 후 `저장 · 연결 테스트` 버튼 동작
- [ ] URL 입력창 Enter 입력 시 저장 동작
- [ ] `해제` 버튼 동작

## 4) 탭 재동기화
- [ ] 기초정보/배당/자산 탭의 `🔄 재동기화` 버튼 동작
- [ ] 동기화 상태 배지(성공/실패/진행중) 업데이트

## 5) Fallback 문구 확인
- [ ] 오늘 데이터가 없을 때 상태 메시지에 fallback 안내가 노출됨
  - 예: `요청일(YYYY.MM.DD) 데이터 없음 → YYYY.MM.DD 사용`

## 6) 기본 정적 체크
- [ ] 전체 JS 문법 체크 통과 (`node --check`)
- [ ] `index.html`, `views_history.js` inline `onclick/onchange/onkeydown` 없음
- [ ] 중복 top-level 함수명 없음
